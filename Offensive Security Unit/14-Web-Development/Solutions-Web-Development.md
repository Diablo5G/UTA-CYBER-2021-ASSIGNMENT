<li>The HTTP protocol version

<p>Answer the following questions about <code>curl</code>:</p>
<ol start="10">
<li>
<p>What are the advantages of using <code>curl</code> over the browser?</p>
<ul>
<li><strong><em><code>curl</code> is a free command line tool with many advantages</em></strong>
<ul>
<li>authentication</li>
<li>HTTP post</li>
<li>SSL connections</li>
<li>proxy support</li>
<li>FTP uploads</li>
<li>Saving URL to file</li>
<li>Downloading</li>
</ul>
</li>
</ul>
</li>
<li>
<p>Which <code>curl</code> option is used to change the request method?</p>
<ul>
<li>The options can be <strong><em>using the <code>-X</code> or <code>--request</code> command-line options</em></strong></li>
</ul>
</li>
<li>
<p>Which <code>curl</code> option is used to set request headers?</p>
<ul>
<li><strong><em><code>-H</code>, <code>--header</code></em></strong></li>
</ul>
</li>
<li>
<p>Which <code>curl</code> option is used to view the response header?</p>
<ul>

</ul>
</li>
<li>
<p>Which request method might an attacker use to figure out which HTTP requests an HTTP server will accept?</p>
<ul>
<li><strong><em><code>GET</code> request because attacker could requests data from a server to figure out which HTTP requests that an HTTP server server will accept.</em></strong></li>

</ul>

<h4><a id="user-content-revisiting-curl" class="anchor" aria-hidden="true" href="#revisiting-curl"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Revisiting curl</h4>
<p>Recall that you used <code>curl</code> to craft different kinds of requests for your <code>curl</code> activity, and that you saw how to use the Chrome extension Cookie-Editor to export and import cookies and swap sessions.</p>
<p>There will be many systems in which you will need to test requests and cookies that will not connect to a browser or browser extension.</p>
<p><code>curl</code> not only allows users to look through headers, send data, and authenticate to servers, but also to save and send cookies through two <code>curl</code> options: <code>--cookie-jar</code> and <code>--cookie</code>.</p>
<p>These two options work exactly like Cookie-Editor, but on the command line.</p>
<ul>
<li>
<p><code>--cookie-jar</code> allows a curl user to save the cookies set within a response header into a text file.</p>
</li>
<li>
<p><code>--cookie</code> allows a user to specify a text file where a cookie is saved, in order to send a request with the cookies embedded in the request header.</p>
</li>
</ul>
<p>Let's look at how we can create a <code>curl</code> command that will log into a web page with a supplied username and password, and also save the server's response that should contain a cookie.</p>
<h4><a id="user-content-logging-in-and-saving-cookies-with-curl" class="anchor" aria-hidden="true" href="#logging-in-and-saving-cookies-with-curl"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Logging In and Saving Cookies with Curl</h4>
<p>If we want to use the <code>curl</code> command to log into an account, <code>Amanda</code>, with the password <code>password</code>, we use the following <code>curl</code> options:</p>
<ul>
<li>
<p><code>curl --cookie-jar ./amandacookies.txt --form "log=Amanda" --form "pwd=password" http://localhost:8080/wp-login.php --verbose</code></p>
</li>
<li>
<p><code>curl</code>: The tool that we are using.</p>
</li>
<li>
<p><code>--cookie-jar</code>: Specifies where we will save the cookies.</p>
</li>
<li>
<p><code>./amandacookies.txt</code>: Location and file where the cookies will be saved.</p>
</li>
<li>
<p><code>--form</code>: Lets us pick the login username and password forms that we set in our user info earlier. In this case it's our username.</p>
</li>
<li>
<p><code>log=Amanda</code>: How WordPress understands and accepts usernames.</p>
</li>
<li>
<p><code>--form</code>: Lets us pick the login username and password forms that we set in our user info earlier. In this case it's our password.</p>
</li>
<li>
<p><code>pwd=password</code>: How WordPress understands and accepts passwords.</p>
</li>
<li>
<p><code>http://localhost:8080/wp-login.php</code>: Our WordPress login page.</p>
</li>
<li>
<p><code>--verbose</code>: Outputs more specific description about the actions the command is taking.</p>
</li>
</ul>
<p>Run the command:  <code>curl --cookie-jar ./amandacookies.txt --form "log=Amanda" --form "pwd=password" http://localhost:8080/wp-login.php --verbose</code></p>
<p>If the site confirms our credentials, it will give us a cookie in return, which <code>curl</code> will save in the cookie jar file <code>./amandacookies.txt</code>.</p>
<p>Now let's look at how to use that saved cookie on a page that requires us to be logged in.</p>
<h4><a id="user-content-using-a-saved-cookie" class="anchor" aria-hidden="true" href="#using-a-saved-cookie"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Using a Saved Cookie</h4>
<p>To use a saved cookie, we use the following <code>curl</code> syntax:</p>
<ul>
<li><code>curl --cookie ./amandacookies.txt http://localhost:8080/wp-admin/users.php</code>
<ul>
<li>
<p><code>curl</code>: The tool that we are using.</p>
</li>
<li>
<p><code>--cookie</code>: Precedes the location of our saved cookie that we want to use.</p>
</li>
<li>
<p><code>./amandacookies.txt</code>: Location and file where the cookies are saved.</p>
</li>
<li>
<p><code>http://localhost:8080/wp-admin/users.php</code>: A page that requires authentication to see properly. Note that we are not going to the login page, because supplying a cookie in this instance assumes that we are already logged in.</p>
</li>
</ul>
</li>
</ul>
<p>Now that we know how to use the <code>curl</code> cookie jar, let's look at what we need to do for this challenge.</p>
